This repository was created from the following human instruction:

I want you to apply this methodology to create a new implementation of the classic game "breakout". I want you to put it in a separate repository that is public in the la3lma github account, check it out into the ~/git/ directory as a subdirectory. The game should be a javascript game, it should have a highscore mechanism for a single user in a single browser, but no multiuser / multi-score. I want you to apply the technology described in this document to that.


In addition I want you to keep a development notebook, in the new repository, as well as the document stack. Use a mermaid diagram that links first to the github issues and then to the github PRs for each of the issues.

Use playwright to capture these things. No need to play games to completion, just enough so that I can see that it works in real time.


Use the description I give here as the instructions for how to make the game. I want that logged in the new repo, I also want this example to be documented using the full stack including the notebook, and included as an example of how the method can be used.
